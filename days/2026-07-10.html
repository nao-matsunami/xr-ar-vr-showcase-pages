今日の重要トピックKey topics today

  1. WebXR Lighting Estimation API Level 1 は 2025-12-11 の Editor's Draft で、XR session の `light-estimation` feature として扱われる。 The WebXR Lighting Estimation API Level 1 is an Editor's Draft dated 2025-12-11 and is requested as the `light-estimation` feature on an XR session.
  2. `XRLightProbe` は環境光推定の入口で、frame ごとの `XRLightEstimate` から primary light direction / intensity と spherical harmonics coefficients を取得する。 `XRLightProbe` is the entry point for environmental lighting, and per-frame `XRLightEstimate` values expose primary light direction, intensity, and spherical harmonics coefficients.
  3. 実装上は現実環境に仮想物を『置く』だけでなく、明るさ、影の向き、色温度が揃うことで AR 合成の違和感を減らせる。 For AR compositing, placing a virtual object is not enough; matching brightness, shadow direction, and color temperature reduces visual mismatch.
  4. Immersive Web の WebXR Samples は hit test、anchors、depth sensing、hand input などを横断的に確認でき、毎日の自作サンプルの参照元として使いやすい。 The Immersive Web WebXR Samples index is useful for checking related topics such as hit testing, anchors, depth sensing, and hand input as references for daily original samples.
  5. Hand Input は 25 joint の手姿勢を扱うため、今後は light / depth / hand の三つを組み合わせて、手前の手に反応する仮想 UI へ発展できる。 Hand Input exposes 25 hand joints, making it a natural next step for combining light, depth, and hands into virtual UI that reacts to foreground fingers.

注目事例と何が面白いかNotable example and why it matters

今日の注目は WebXR Lighting Estimation です。面白い点は、AR の説得力が hit test や anchor の位置合わせだけで決まらず、環境光に合わせて仮想物の明るさ、影、色味を変えることで大きく変わることです。仕様では `XRLightProbe` と `XRLightEstimate` を通じて primary light と spherical harmonics を扱うため、単なる UI slider ではなく、描画パイプラインへ入る光情報として考える必要があります。

Today's notable reference is WebXR Lighting Estimation. The interesting point is that AR believability is not determined only by hit-test placement or anchors; matching virtual brightness, shadows, and color to the real environment matters. The specification exposes primary light and spherical harmonics through `XRLightProbe` and `XRLightEstimate`, so lighting should be treated as render-pipeline data rather than just UI state.

表現・インタラクションのアイデアExpression and interaction idea

light probe を見えない内部値にせず、primary direction、ambient intensity、temperature を小さな meter として見せると、AR 合成の状態が理解しやすくなります。今日のサンプルでは、Window / Desk / Gallery のプリセットと slider、画面タップによる光源方向変更を用意し、同じ仮想カードが環境推定値だけで違う質感に見えるようにしました。

Instead of hiding the light probe as internal data, showing primary direction, ambient intensity, and temperature as small meters makes AR compositing more legible. Today's sample includes Window, Desk, and Gallery presets plus sliders and tap-based light direction changes, so the same virtual card changes its material feeling only from estimated environment values.

制作に使えそうな技術メモProduction note

実 WebXR では `navigator.xr.requestSession("immersive-ar", { optionalFeatures: ["light-estimation"] })` のように feature を要求し、session から light probe を作成して、frame ごとに light estimate を読む流れになります。primary light direction / intensity は directional light と shadow に、spherical harmonics coefficients は ambient diffuse や image based lighting の近似に渡すと使いやすいです。

In real WebXR, the session can request a feature such as `optionalFeatures: ["light-estimation"]`, create a light probe from the session, and read a light estimate each frame. Primary light direction and intensity map well to directional lights and shadows, while spherical harmonics coefficients are useful for ambient diffuse or approximate image-based lighting.
