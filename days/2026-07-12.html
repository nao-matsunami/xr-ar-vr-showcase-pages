今日の重要トピックKey topics today

  1. WebXR Layers API Level 1 の Editor's Draft は 2026-06-12 更新で、WebXR session 内の projection / quad / cylinder / equirect / cube などの composition layer を扱う。 The WebXR Layers API Level 1 Editor's Draft was updated on 2026-06-12 and covers composition layers such as projection, quad, cylinder, equirect, and cube layers within WebXR sessions.
  2. Layers は `layers` feature descriptor を immersive-vr / immersive-ar session で要求し、`XRWebGLBinding` や `XRMediaBinding` から layer を作成して render state に追加する流れになっている。 Layers are requested through the `layers` feature descriptor in immersive-vr or immersive-ar sessions, then created through `XRWebGLBinding` or `XRMediaBinding` and added to render state.
  3. Composition layer は text-optimized / graphics-optimized の quality を持ち、HUD や地図、文字 UI の可読性を eye buffer だけに依存しない形で高められる。 Composition layers can use text-optimized or graphics-optimized quality, improving readability for HUDs, maps, and text UI without relying only on the eye buffer.
  4. Depth Sensing と Hand Input を重ねると、手が UI より手前にある時だけ遮蔽し、奥にある仮想物を薄くするような空間 UI の整合性を作りやすい。 Combining Depth Sensing and Hand Input makes spatial UI coherence easier: content can hide only when a hand is physically in front of the UI, while farther virtual content can fade.
  5. Lighting Estimation の考え方を足すと、手元の layer の影、明るさ、選択 feedback を環境光に合わせて調整できる。 Adding Lighting Estimation ideas lets hand-adjacent layers adjust shadows, brightness, and selection feedback to the surrounding light.

注目事例と何が面白いかNotable example and why it matters

今日の注目は WebXR Layers API です。面白い点は、通常の projection layer だけでなく、quad layer や cylinder layer のような composition layer を compositor 側で扱い、テキストや地図のような細かい UI を高品質に保てることです。仕様上は `XRCompositionLayer.opacity`、`quality`、`needsRedraw` などがあり、動き続ける 3D scene と、読みやすい固定 UI を分けて考えられます。

Today's notable reference is the WebXR Layers API. The interesting part is that composition layers such as quad and cylinder layers can be handled by the compositor rather than relying only on the projection layer, helping detailed UI like text or maps remain legible. The specification includes concepts such as `XRCompositionLayer.opacity`, `quality`, and `needsRedraw`, letting authors separate a moving 3D scene from readable fixed UI.

表現・インタラクションのアイデアExpression and interaction idea

Layers の利点を UI の見やすさだけで終わらせず、Hand Input と Depth Sensing の観察点を混ぜると、手が quad layer より手前に入った時だけ UI を自然に隠す表現になります。今日のサンプルでは pointer を手として扱い、hand depth と layer depth の差から遮蔽率を計算し、pinch が強い時だけ hover から select に変わるようにしました。

Layers become more than a readability tool when combined with Hand Input and Depth Sensing: UI can be hidden naturally only when a hand is in front of the quad layer. Today's sample treats the pointer as a hand, computes occlusion from the hand/layer depth difference, and changes hover into select only when pinch is strong.

制作に使えそうな技術メモProduction note

実装時は `navigator.xr.requestSession("immersive-ar", { optionalFeatures: ["layers", "depth-sensing", "hand-tracking", "light-estimation"] })` のように機能を分けて要求し、layer は `XRWebGLBinding` から作ります。遮蔽は depth texture または CPU depth data と hand joint の位置を同じ reference space にそろえ、UI layer の矩形内で前後関係を評価するのが基本になります。

In a real implementation, features can be requested separately, for example `navigator.xr.requestSession("immersive-ar", { optionalFeatures: ["layers", "depth-sensing", "hand-tracking", "light-estimation"] })`, while layers are created from `XRWebGLBinding`. For occlusion, depth textures or CPU depth data and hand joint positions should be resolved into the same reference space, then tested against the UI layer rectangle.
