今日の重要トピックKey topics today

  1. WebXR Layers API Level 1 は 2026-06-12 版 Editor's Draft で、equirect / cube / media layer と depth sorting の設計点を含んでいる。 The WebXR Layers API Level 1 Editor's Draft dated 2026-06-12 includes equirect, cube, media layers, and depth-sorting design points.
  2. XREquirectLayer は equirectangular 形式のデータを球の内側に貼る layer で、360 度映像や写真の背景に向く。 XREquirectLayer maps equirectangular data onto the inside of a sphere, making it suitable for 360-degree video and photo backgrounds.
  3. XRMediaBinding には HTMLVideoElement から quad / cylinder / equirect の media layer を作る入口があり、動画を通常の projection layer から分離しやすい。 XRMediaBinding provides entry points for creating quad, cylinder, and equirect media layers from an HTMLVideoElement, making video easier to separate from the regular projection layer.
  4. depth-sorted-layers は projection / quad / cylinder layer の奥行き順制御に関わるが、equirect / cube のような環境 layer は別の扱いとして考える必要がある。 depth-sorted-layers affects depth ordering for projection, quad, and cylinder layers, while environmental layers such as equirect and cube layers need separate treatment.
  5. 字幕や注釈は背景メディアと同じテクスチャに焼き込まず、quad layer として分けると読みやすさ、透明度、更新頻度を独立して調整できる。 Captions and annotations should often be separated into quad layers rather than baked into the background media, so legibility, opacity, and update frequency can be tuned independently.

注目事例と何が面白いかNotable example and why it matters

今日の注目は WebXR Layers API の equirect / media layer です。面白い点は、360 度映像や写真を scene の通常描画に混ぜるのではなく、環境 layer として compositor 側に渡し、字幕や注釈を別の quad layer として重ねられることです。映像は滑らかに動かし、文字は読みやすい解像度と安定した位置を保つ、という分業がしやすくなります。

Today's notable reference is the equirect and media layer area of the WebXR Layers API. The interesting part is that 360-degree video or photography can be handed to the compositor as an environment layer instead of being mixed into normal scene rendering, while captions and annotations can sit in separate quad layers. This makes it easier to let video move smoothly while text keeps readable resolution and a stable position.

表現・インタラクションのアイデアExpression and interaction idea

360 背景は『場』として広く動かし、字幕と注釈は『読ませる layer』として視野内に固定する構成が有効です。今日のサンプルでは canvas をドラッグして viewer yaw / pitch を動かし、Tour / Sports / Lab の 3 種類で背景メディアの動きと注釈の乗り方を比較できます。caption depth を近づけすぎると、字幕は目立つ一方で視界の圧迫感が増える、という設計上のトレードオフも見えます。

A useful composition is to let the 360 background behave as the surrounding place, while captions and annotations stay as readable layers inside the view. Today's sample lets you drag the canvas to change viewer yaw and pitch, then compare Tour, Sports, and Lab modes for how background motion and annotations combine. Pulling caption depth too close makes captions more prominent but also more visually intrusive.

制作に使えそうな技術メモProduction note

実 WebXR では `navigator.xr.requestSession("immersive-vr", { optionalFeatures: ["layers"] })` で layers を有効にし、動画を使う場合は `new XRMediaBinding(session).createEquirectLayer(video, init)` のような流れを検討します。字幕や注釈は `XRWebGLBinding` 由来の quad layer、または media / canvas 由来の別 layer として扱い、`opacity`、`quality`、`blendTextureSourceAlpha`、`layers` 配列の順序を明示的に管理するのが要点です。

In a real WebXR implementation, layers can be enabled through `navigator.xr.requestSession("immersive-vr", { optionalFeatures: ["layers"] })`, and video-backed environments can be explored through a flow such as `new XRMediaBinding(session).createEquirectLayer(video, init)`. Captions and annotations should be handled as separate quad layers from `XRWebGLBinding`, or as separate media/canvas-backed layers, with explicit control over `opacity`, `quality`, `blendTextureSourceAlpha`, and the `layers` array order.

明日以降ウォッチする対象Watch next

次は WebXR Layers の redraw / needsRedraw と、静的 layer をどのタイミングで更新すべきかを見たいです。字幕や注釈は毎フレーム描き直す必要がない場合も多く、動画 layer、静的注釈 layer、動的ポインタ layer の更新頻度を分けると、電力と視覚安定性の両方に効きます。あわせて Space Warp と foveation も追跡対象にします。

Next, it is worth watching redraw / needsRedraw behavior in WebXR Layers and when static layers should be updated. Captions and annotations often do not need to be redrawn every frame, so separating update rates across video layers, static annotation layers, and dynamic pointer layers can help both power use and visual stability. Space Warp and foveation are also worth tracking.
