今日の重要トピックKey topics today

  1. WebXR Hit Test Module は 2025-12-11 の W3C Working Draft として、現実空間ジオメトリに対する ray intersection を扱う。 The WebXR Hit Test Module is a W3C Working Draft dated 2025-12-11 and focuses on ray intersections against real-world geometry.
  2. Hit Test の公式サンプルは、実空間の床や面に仮想オブジェクトを置く最小導線を確認する入口になる。 The official Hit Test sample is a compact entry point for placing virtual objects on real-world floors or surfaces.
  3. Anchors Module は `XRAnchor` と `anchorSpace`、persistent handle の考え方を定義しており、配置後の保持を設計する軸になる。 The Anchors Module defines `XRAnchor`, `anchorSpace`, and persistent handles, which guide how placed content can be retained.
  4. DOM Overlay は immersive session 中の 2D UI を扱い、`beforexrselect` で DOM 操作と XR world select の衝突を分けられる。 DOM Overlay covers 2D UI during immersive sessions and uses `beforexrselect` to separate DOM interactions from XR world select events.
  5. 今日の自作サンプルは、本物の XR セッション前に ray、reticle、anchor list、drift の関係だけをブラウザ上で切り出した。 Today's original sample isolates ray, reticle, anchor list, and drift behavior in the browser before using a real XR session.

注目事例と何が面白いかNotable example and why it matters

今日の注目は WebXR Hit Test の公式サンプルです。面白い点は、空間認識そのものを大きく見せるのではなく、ray が実世界の面に当たった一点を reticle として示し、そこにユーザーの決定を重ねる構造が非常に小さいことです。これに Anchors の発想を足すと、置いた瞬間だけでなく、置いた後にどれだけ信用できるかを UI として見せる必要が出てきます。

The notable reference today is the official WebXR Hit Test sample. What matters is how small the interaction structure is: show the point where a ray hits a real-world surface as a reticle, then let the user commit to that point. Adding the Anchors idea shifts the question from placement alone to how confidently placed content can persist afterward.

表現・インタラクションのアイデアExpression and interaction idea

reticle を単なる照準ではなく、面の信頼度、ray の角度、anchor の数を同時に示す小さな計器として扱うと、AR らしい配置前の迷いを表現できます。今日は床面 grid と中心 ray、クリック配置、drift ボタンによる追跡ノイズだけに絞り、実装より先に『置けそう／置くべきでない』のフィードバック設計を確認しました。

Treating the reticle not just as a cursor but as a compact instrument for surface confidence, ray angle, and anchor count makes the uncertainty before AR placement visible. Today's prototype keeps only a floor grid, center ray, click placement, and drift button so the feedback design can be tested before a full implementation.

制作に使えそうな技術メモProduction note

実 WebXR では session request 時に `hit-test` や `dom-overlay` を optionalFeatures / requiredFeatures として扱い、XRFrame から hit test result の pose を取り、必要なら anchor を作る流れになります。今回の HTML 版は canvas 上で疑似 floor projection を描くだけですが、UI 分離としては DOM Overlay の考え方に近く、2D 操作面と空間配置面を分けて検証できます。

In real WebXR, a session would request features such as `hit-test` and `dom-overlay`, read hit-test poses from XRFrame, and create anchors where needed. This HTML prototype only draws a simulated floor projection on canvas, but its UI separation mirrors the DOM Overlay idea by separating the 2D control surface from spatial placement.
