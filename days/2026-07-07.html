The public XR archive policy was corrected so that only self-made prototypes appear as dated cards. Existing official demos will still be researched, but they will remain as reference links inside the report rather than public archive entries.

注目事例と何が面白いかNotable reference and why it matters

参考元として見た WebXR Layers 系の発想で重要なのは、HUD や補助面を 3D シーン内メッシュと同じ扱いにせず、読みやすさのために別の層として考えることです。今回はその考え方だけを抽出し、実装依存の少ない HTML/CSS/JS で疑似的に試しました。

The key idea borrowed from WebXR Layers references is to treat HUD and support surfaces as a separate readability layer rather than ordinary scene meshes. This prototype extracts only that idea and tests it in dependency-light HTML/CSS/JS.

表現・インタラクションのアイデアExpression and interaction idea

ポインタを疑似 head pose と見なし、前景 HUD、中景説明面、奥の基準面で反応速度と視差量を変えると、XR での情報の優先順位を Web 上でもかなり早く比較できます。次は gaze 固定、hand ray、hit test で面が差し替わる版へ広げられます。

By treating the pointer as a mock head pose and varying response speed and parallax across foreground HUD, mid-layer prompts, and deep anchor planes, XR information hierarchy can be tested quickly on the web. The next step would be versions driven by gaze fixation, hand rays, or hit-test results.

制作に使えそうな技術メモProduction note

今回は WebXR 実装に入る前の構造確認として、CSS perspective と pointer tracking のみで層分離を検証しています。利点は、HMD や端末依存なしに、レイヤー距離、面積、読解距離、アニメーション速度を先に詰められることです。

This prototype checks layered structure before moving into full WebXR by using only CSS perspective and pointer tracking. The advantage is that layer spacing, panel size, reading distance, and animation speed can be refined without requiring an HMD or device-specific setup.

明日以降ウォッチする対象Watch next

次は WebXR 本体に寄せて、enter immersive 前後で HUD を切り替える最小構成、もしくは AR 的な情報面のアンカー切り替えを確認したいです。加えて、XR での text readability と panel contrast の実測条件も溜めていきます。

Next, the work should move closer to actual WebXR by testing a minimal HUD switch around enter-immersive, or an AR-like anchor-switching information surface. It is also worth collecting practical conditions for text readability and panel contrast in XR.

試すならこれIf you try one thing

まずは今回の `Spatial HUD Depth Stack` を見て、前景面・中景面・奥行き面のどこに何の情報を置くと読みやすいかを整理するのが良いです。特に『説明は奥に、即時操作は手前に』というルールがどこまで有効かを試せます。

Start by using the current Spatial HUD Depth Stack to map which information belongs to foreground, middle, and depth planes. In particular, it helps test how far the rule of 'instructions deeper, immediate controls closer' remains effective.
