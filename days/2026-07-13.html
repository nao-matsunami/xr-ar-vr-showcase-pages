今日の重要トピックKey topics today

  1. WebXR Layers API Level 1 は 2026-06-12 版 Editor's Draft で、projection / quad / cylinder / equirect / cube などの composition layer を定義している。 The WebXR Layers API Level 1 Editor's Draft dated 2026-06-12 defines composition layers such as projection, quad, cylinder, equirect, and cube layers.
  2. XRCylinderLayer は視野をゆるく包む UI やメディア面に向いており、平面 HUD より端の角度と読みやすさを観察しやすい。 XRCylinderLayer is useful for UI or media surfaces that gently wrap around the field of view, making edge angle and readability easier to study than with flat HUDs.
  3. 仕様では layer の `quality` に text-optimized / graphics-optimized があり、文字、地図、字幕のような情報を通常の eye buffer から分けて扱える。 The specification includes layer `quality` values such as text-optimized and graphics-optimized, allowing text, maps, and captions to be treated separately from the regular eye buffer.
  4. stereo-left-right や stereo-top-bottom の layout は既存の stereo media を少ない draw call で扱うための考え方として重要。 stereo-left-right and stereo-top-bottom layouts matter as a way to handle existing stereo media with fewer draw calls.
  5. 複数 layer の depth sorting は、近いカード、曲面 UI、背景 scene が重なる時に破綻が見えやすく、実装前に最小サンプルで確認する価値がある。 Depth sorting between multiple layers becomes visibly fragile when a near card, curved UI, and background scene overlap, so it is worth checking in a minimal sample before a full implementation.

注目事例と何が面白いかNotable example and why it matters

今日の注目は WebXR Layers API の cylinder layer です。面白い点は、平面の quad layer だけでなく、視野に沿って湾曲する layer を compositor 側で扱えることです。文字、地図、字幕、動画パネルのような情報は、3D scene と同じ projection layer に混ぜるより、曲面 composition layer として分けた方が読みやすさと描画更新頻度を設計しやすくなります。

Today's notable reference is the cylinder layer in the WebXR Layers API. The interesting part is that the compositor can handle not only flat quad layers but also layers curved around the field of view. Information surfaces such as text, maps, captions, and video panels can be easier to design for readability and update frequency when separated into curved composition layers instead of being mixed into the same projection layer as the 3D scene.

表現・インタラクションのアイデアExpression and interaction idea

曲面 UI は『読むための空間』として使えます。viewer yaw を変えると中央の情報は読みやすく、端の情報は角度がつくため、重要度に応じて情報を中央へ寄せる設計が必要です。今日のサンプルでは Text / Map / Video / Depth のモードを切り替え、同じ曲面 layer に載せる情報種別で、端の歪み、stereo offset、手前カードとの重なり方を比較できます。

Curved UI can act as a space built for reading. As viewer yaw changes, central information stays readable while edge content becomes angled, so important information should be biased toward the center. Today's sample switches between Text, Map, Video, and Depth modes, letting the same curved layer compare edge skew, stereo offset, and overlap with a nearer card across different information types.

制作に使えそうな技術メモProduction note

実 WebXR では `navigator.xr.requestSession("immersive-vr", { optionalFeatures: ["layers"] })` のように layers を要求し、`XRWebGLBinding` または `XRMediaBinding` から cylinder / quad / media layer を作る流れになります。曲面 UI は半径、centralAngle、aspectRatio、space、transform を決める必要があり、文字 layer は `quality = "text-optimized"`、動画や図版は `graphics-optimized` を試す候補になります。

In a real WebXR implementation, layers can be requested with something like `navigator.xr.requestSession("immersive-vr", { optionalFeatures: ["layers"] })`, then cylinder, quad, or media layers are created from `XRWebGLBinding` or `XRMediaBinding`. Curved UI needs decisions around radius, centralAngle, aspectRatio, space, and transform; text layers are candidates for `quality = "text-optimized"`, while video or graphics can test `graphics-optimized`.
