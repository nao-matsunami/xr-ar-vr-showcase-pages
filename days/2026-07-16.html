今日の重要トピックKey topics today

  1. WebXR Layers API Level 1 の 2026-06-12 版 Editor's Draft では、composition layer の再投影、遅延低減、Space Warp が整理されている。 The WebXR Layers API Level 1 Editor's Draft dated 2026-06-12 describes composition layer reprojection, latency reduction, and Space Warp.
  2. XRProjectionLayer.deltaPose は、前フレームからビューに影響した application-applied transform の差分を表し、Space Warp feature を要求していない場合は null または no-op になり得る。 XRProjectionLayer.deltaPose represents the incremental application-applied transform since the previous frame that affects the view, and may be null or a no-op when Space Warp support was not requested.
  3. deltaPose の値は requestAnimationFrame の終端で自動 reset されず、人工 locomotion などで transform が変わったときに更新し、変化がないフレームでは null または identity に戻す設計が求められる。 deltaPose is not automatically reset at the end of requestAnimationFrame, so authors should update it when transforms change and set null or identity when no application transform occurred.
  4. MDN は XRProjectionLayer を observer の全視野を埋め、native frame rate 近くで refresh される layer と説明し、layers render state への追加には XRSession.updateRenderState() を使う。 MDN describes XRProjectionLayer as a full-view layer refreshed close to the native frame rate, with XRSession.updateRenderState() used to add layers to render state.
  5. 低頻度で app が描画する layer と高頻度に compositor が補間する layer を分けると、可読 UI や背景の描画負荷を抑えつつ、頭部移動時の体感だけを保つ余地が出る。 Separating low-rate app-rendered layers from high-rate compositor synthesis can reduce rendering cost while preserving perceived stability during head movement.

注目事例と何が面白いかNotable example and why it matters

今日の注目は WebXR Layers API の Space Warp と `XRProjectionLayer.deltaPose` です。面白い点は、application が毎フレーム完全に描けない場合でも、前回描画した projection layer に対して『このフレームまでに world がどれだけ動いたか』を差分として compositor に渡すことで、欠けた中間フレームを合成する余地があることです。昨日の static layer redraw は『描かないで済ませる』話でしたが、今日は『描けない間の見え方を補う』話です。

Today's notable area is Space Warp and `XRProjectionLayer.deltaPose` in the WebXR Layers API. The interesting part is that even when the application cannot fully render every frame, it can describe how the world changed since the previous frame so the compositor has a chance to synthesize missing frames. Yesterday's static layer redraw was about avoiding unnecessary drawing; today is about compensating visual motion while drawing less often.

表現・インタラクションのアイデアExpression and interaction idea

空間 UI では、背景や大きな projection layer を 24Hz 程度でしか描けない状況でも、頭部 yaw の差分を deltaPose として扱えば、compositor 側の再投影で体感のずれを減らせる可能性があります。今日のサンプルでは、cyan が最後に app が描いた pose、green が deltaPose 風に補正した pose、薄い white が実際の現在 pose です。Render rate を下げ、Warp off にすると、低頻度描画の遅れが視覚的に分かります。

For spatial UI, even if a background or large projection layer can only be rendered at around 24 Hz, treating head-yaw changes as deltaPose can reduce perceived drift through compositor reprojection. In today's sample, cyan is the last app-rendered pose, green is the deltaPose-style compensated pose, and faint white is the current pose. Lowering render rate and toggling Warp off makes the low-rate delay visible.

制作に使えそうな技術メモProduction note

実装時は `navigator.xr.requestSession("immersive-vr", { optionalFeatures: ["layers", "space-warp"] })` のように Space Warp を要求できるかを確認し、`XRWebGLBinding.createProjectionLayer()` で projection layer を作り、`XRSession.updateRenderState({ layers: [projectionLayer, ...] })` に載せる流れを検討します。`deltaPose` は前フレームからの application-applied transform の差分なので、頭部 pose そのものではなく、人工 locomotion、world origin の補正、低頻度 world update の差分として扱うのが要点です。

A real implementation should check whether Space Warp can be requested with a shape such as `navigator.xr.requestSession("immersive-vr", { optionalFeatures: ["layers", "space-warp"] })`, create a projection layer with `XRWebGLBinding.createProjectionLayer()`, and place it in `XRSession.updateRenderState({ layers: [projectionLayer, ...] })`. `deltaPose` is an application-applied transform difference since the previous frame, so the key is to treat it as artificial locomotion, world-origin correction, or low-rate world-update delta rather than the raw head pose itself.

今日のミニ試作候補Today's mini prototype candidate

採用した候補は `Delta Pose Space Warp Lab` です。実機 API を直接使わず、projection layer が低頻度でしか描けない状態を canvas 上で擬似化し、最後に描いた pose、現在 pose、deltaPose 風の補正 pose を重ねて表示します。次に発展させるなら、手前の pointer/hand layer を追加し、補正された projection layer と未補正 foreground layer の depth mismatch を観察します。

The selected prototype is Delta Pose Space Warp Lab. Without directly using hardware XR APIs, it simulates a projection layer that renders at a low rate and overlays the last rendered pose, the current pose, and a deltaPose-style compensated pose. A natural next step is adding a foreground pointer or hand layer to inspect depth mismatch between a compensated projection layer and an uncompensated foreground layer.
