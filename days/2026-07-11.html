今日の重要トピックKey topics today

  1. WebXR Hand Input Module Level 1 は 2024-06-05 の Working Draft / Editor's Draft として公開され、articulated hand pose を WebXR Device API に追加する。 The WebXR Hand Input Module Level 1 is available as a 2024-06-05 Working Draft / Editor's Draft and adds articulated hand poses to the WebXR Device API.
  2. セッションで `hand-tracking` feature descriptor を要求すると、対応する physical hand input source から `XRInputSource.hand` を通じて `XRHand` を取得できる。 When a session requests the `hand-tracking` feature descriptor, compatible physical hand input sources expose an `XRHand` through `XRInputSource.hand`.
  3. `XRHand` は 25 個の joint を持ち、`wrist` から各指の metacarpal / phalanx / tip までの名前と順序が仕様で定義されている。 `XRHand` contains 25 joints, with names and order defined from `wrist` through each finger's metacarpal, phalanx, and tip joints.
  4. frame loop では `XRFrame.getJointPose()` や `fillJointRadii()` により、joint pose と指先 collision に使える radius を読む設計になっている。 The frame loop reads joint poses and collision-friendly radii through methods such as `XRFrame.getJointPose()` and `fillJointRadii()`.
  5. 手入力 UI は pinch だけでなく、指先の近接、hover、押し込み、誤認識時の feedback を別々に可視化すると調整しやすい。 Hand UI is easier to tune when pinch, fingertip proximity, hover, press depth, and recognition feedback are visualized separately.

注目事例と何が面白いかNotable example and why it matters

今日の注目は WebXR Hand Input です。面白い点は、controller のボタンイベントではなく、手首と各指の joint pose を frame ごとに取得し、接近、pinch、衝突判定、手モデル描画を同じ skeleton から組み立てられることです。仕様では `XRHand.size` が 25 で、各 joint 名と順序が固定されるため、UI 側は index-tip、thumb-tip、wrist などを安定して参照できます。

Today's notable reference is WebXR Hand Input. The interesting part is that interaction can be built from per-frame wrist and finger joint poses rather than controller button events, so proximity, pinch, collision, and hand rendering can all come from the same skeleton. The specification fixes `XRHand.size` at 25 and defines stable joint names and order, allowing UI code to consistently target joints such as index-tip, thumb-tip, and wrist.

表現・インタラクションのアイデアExpression and interaction idea

手の skeleton をそのまま見せるだけでなく、index-tip から対象パネルへの距離線、thumb-tip との pinch 距離、tip radius を同時に出すと、空間 UI が hover なのか select なのかを説明しやすくなります。今日のサンプルでは Open / Pinch / Reach の preset、curl / spread / depth の slider、drag で手全体の位置移動を用意し、近接と pinch を分けて観察できるようにしました。

Instead of only showing a hand skeleton, drawing the distance from the index tip to a target panel, the pinch distance to the thumb tip, and the fingertip radius makes hover versus select states easier to understand. Today's sample includes Open, Pinch, and Reach presets, curl/spread/depth sliders, and drag-based hand movement so proximity and pinch can be inspected separately.

制作に使えそうな技術メモProduction note

実 WebXR では `navigator.xr.requestSession("immersive-vr", { optionalFeatures: ["hand-tracking"] })` のように feature を要求し、input source の `hand` から `XRJointSpace` を列挙します。毎 frame で `getJointPose(jointSpace, baseSpace)` を呼び、tip joint の位置と `XRJointPose.radius` を UI hit area や collision volume に変換すると、controller ray とは違う近接型インタラクションを作れます。

In real WebXR, a session can request a feature such as `optionalFeatures: ["hand-tracking"]`, then enumerate `XRJointSpace` values from an input source's `hand`. Each frame can call `getJointPose(jointSpace, baseSpace)` and convert tip joint positions plus `XRJointPose.radius` into UI hit areas or collision volumes, enabling proximity interactions that differ from controller rays.
